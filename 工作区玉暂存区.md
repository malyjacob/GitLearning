# 工作区和暂存区
工作区就是电脑里能看到的目录

版本库(repository)
工作区有个隐藏的目录`.git`,这个不算工作区,而是git的版本库.

git这个版本库可以分为两大部分:

`stage`(`index`)==暂存区 <br/>
`master`为版本库第一个分支, 有一个指针`HEAD`指向`master`.

之前我们的操作可以总结为: <br/>
第一步是`git add` 把新文件或者文件的修改添加到暂存区; <br/>
第二部是`git commit`提交更改,也就是把暂存区里的所有内容提交到当前分支里去. <br/>

一旦提交后, 如果你又没有对工作区做什么修改, 那么运行`git status` 就是"干净的".
